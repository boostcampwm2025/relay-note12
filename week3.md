## 서론
첫 만남 후, 아이스 브레이킹 토크에서 챌린지 과정에 대한 대화를 하던 중 <br/>
학습, 구현 내용을 정리하면서 문서화하고 복습하는 것과 <br/>
체력, 컨디션 관리도 중요한 것 같다는 대화를 나누며 아이스 브레이킹을 했는데, <br/>
팀원 모두가 해당 부분에 대해 공감을 했습니다. <br/>

아이스 브레이킹을 끝내고 주제를 확인을 했는데, <br/>
주제가 **개발 이외의 챙겨야 할 부분을 AI에게 맡기자** 여서 <br/>
팀원 분들과 회의를 진행을 했습니다.

## 공통된 의견
학습 자체를 AI에게 맡기는 것이 아닌 문서화 틀을 잡는데 활용하거나 <br/>
성장의 출발점으로 활용하는데에 팀원들이 동의하였습니다. <br/>
이에 개발 외적으로 AI로부터 개발 이외에 성장에 도움 받을 수 있는 부분을 찾아보기로 했습니다. <br/>
학습 자체를 맡기는 것이 AI를 활용하여 학습의 출발점을 제공 받거나 <br/>
건강, 컨디션 관리에 도움 받을 수 있는 부분을 찾아보기로 했습니다.

## 요청
- 요즈음 장시간 책상에앉아서 개발공부하면서 키보드와 마우스를 사용했더니 오른쪽팔꿈치 통증이 심해
3분스트레칭 사이클과 책상에 앉아있을때의 올바른 자세, 효과등을 알려줘

## 답변
장시간 책상에 앉아 개발 공부를 하며 오른쪽 팔꿈치 통증이 발생하는 경우, 스트레칭과 올바른 자세를 유지하는 것이 중요합니다.

**3분 스트레칭 사이클**

1. **손목 스트레칭** (30초)
   - 팔꿈치를 구부리고 손목을 뒤로 젖혀서 손바닥이 천장을 향하게 합니다. 
   - 반대 손으로 손끝을 잡고 부드럽게 당겨줍니다.

2. **어깨 스트레칭** (30초)
   - 팔꿈치를 구부려 어깨 높이에서 팔을 앞쪽으로 뻗고, 
   - 반대 손으로 팔꿈치를 잡고 몸 쪽으로 당깁니다. 어깨 근육을 부드럽게 늘려줍니다.

3. **팔꿈치 회전** (30초)
   - 팔꿈치를 직각으로 구부리고 손목을 위로 향하게 한 후, 팔꿈치를 천천히 원을 그리며 회전시킵니다.
   - 시계방향, 반시계방향 각각 15초씩.

4. **목 스트레칭** (30초)
   - 한 손으로 머리를 옆으로 눌러 목을 기울여서 어깨 근육을 스트레칭합니다. 
   - 반대쪽도 반복합니다.

5. **상체 비틀기** (30초)
   - 팔꿈치를 가슴 높이로 구부리고, 상체를 왼쪽으로 비틀며 오른팔꿈치를 왼쪽으로 돌려 스트레칭합니다.
   - 반대쪽도 반복합니다.

6. **손목 회전** (30초)
   - 손목을 가볍게 원을 그리며 회전시킵니다. 
   - 손목을 시계방향, 반시계방향으로 각각 15초씩 회전시켜 줍니다.

---

**책상에 앉을 때의 올바른 자세**

1. **모니터 위치**: 
   - 모니터 상단이 눈높이에 맞게 배치하고, 화면과의 거리는 약 50-70cm 정도가 적당합니다.
   
2. **목과 척추의 정렬**: 
   - 머리, 목, 척추가 일직선을 이루도록 하며, 고개를 숙이지 않고 모니터를 직시합니다.
   
3. **팔꿈치 각도**: 
   - 팔꿈치는 90도를 유지하면서 키보드와 마우스를 사용합니다.
   - 팔꿈치가 너무 펴지거나 구부러지지 않도록 주의합니다.
   
4. **다리 위치**: 
   - 다리는 평평하게 바닥에 두고 무릎이 90도 각도를 이루도록 합니다.
   - 발받침대를 사용하는 것도 좋은 방법입니다.
   
5. **키보드와 마우스**: 
   - 키보드는 팔꿈치 높이에 맞게 배치하고, 마우스는 팔꿈치가 편안하게 구부러진 상태에서 사용할 수 있도록 합니다.

**스트레칭 및 자세 개선 효과**

- **통증 완화**: 자주 스트레칭을 하면 근육이 이완되어 통증이 완화될 수 있습니다.
- **근육 피로 감소**: 지속적으로 스트레칭을 하면 근육이 긴장을 풀고, 피로감을 줄여줍니다.
- **혈액 순환 개선**: 스트레칭은 혈액 순환을 도와 근육에 산소와 영양분이 공급되는 데 도움을 줍니다.
- **자세 개선**: 올바른 자세를 유지하면 근육과 관절에 무리가 가지 않으므로 장시간 작업 후에도 피로감이 덜합니다.

# 서버 코드: Server 클래스

### 주요 함수

#### `createServer()`
- **역할**: TCP 서버를 생성하고 클라이언트의 연결을 수락하여 데이터를 처리하는 서버를 구성합니다.
- **주요 변수**
  - **this.connections**: 클라이언트의 연결 정보를 저장하는 `Map`.
  - **this.catalog**: 상품 정보를 저장하는 객체.

#### `commandHandler(jsonData, socket)`
- **역할**: 클라이언트에서 받은 명령어를 처리합니다.
- **주요 변수**
  - **jsonData.command**: 클라이언트가 보낸 명령어.
  - **jsonData.content**: 명령어와 함께 전달된 내용.

#### `loginCommand(content, socket)`
- **역할**: 클라이언트의 로그인 요청을 처리합니다.
- **주요 변수**
  - **content**: 클라이언트가 보낸 아이디.
  - **socket**: 클라이언트와 연결된 소켓.
- ***주의 사항***: 로그인 시 아이디 형식을 검증하며, 이미 사용 중인 아이디는 재사용할 수 없습니다.

#### `logoutCommand(content, socket)`
- **역할**: 클라이언트의 로그아웃 요청을 처리합니다.
- **주요 변수**
  - **content**: 클라이언트가 보낸 아이디.
  - **socket**: 클라이언트와 연결된 소켓.
- ***주의 사항***: 로그인 상태에 없는 사용자는 로그아웃할 수 없습니다.

#### `catalogCommand(content, socket)`
- **역할**: 클라이언트의 상품 목록 조회 요청을 처리합니다.
- **주요 변수**
  - **content**: 클라이언트가 보낸 요청 내용.

# 클라이언트 코드: ClientCustomer 클래스

### 주요 함수

#### `createConnection()`
- **역할**: 서버와의 TCP 연결을 설정합니다.
- **주요 변수**
  - **this.client**: 서버와 연결된 소켓.
- ***주의 사항***: 서버에 연결될 때, 연결 오류 발생 시 안내 메시지를 출력합니다.

#### `userInput()`
- **역할**: 사용자 입력을 받아 서버로 명령어를 전송합니다.
- **주요 변수**
  - **input**: 사용자가 입력한 아이디.
- ***주의 사항***: 아이디 형식이 올바르지 않으면 경고 메시지를 출력하고 종료합니다.

#### `userOutput(data)`
- **역할**: 서버에서 받은 응답을 출력합니다.
- **주요 변수**
  - **data**: 서버에서 받은 응답 데이터.
- ***주의 사항***: 응답에서 상품 목록이 있을 경우 출력하며, JSON 응답 형식에 맞지 않으면 오류 메시지를 출력합니다.
